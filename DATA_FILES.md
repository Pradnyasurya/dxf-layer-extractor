# DXF Layer Validator - Data Files

This document describes the JSON data files used by the application.

## Validation Rules

### odisha_layers.json (133KB)
**Primary validation rules** for the DXF layer validator.
- Contains ~500 layer validation rules
- Each rule has: Feature, Layer Name, Color Code, Requirement, Type, Drawing Guidelines
- Layer patterns use `_n` notation (e.g., `BLK_n_COVERED_AREA` matches `BLK_1_COVERED_AREA`, `BLK_2_COVERED_AREA`, etc.)
- **This is the main rules file used by the application**

### ppa_layers.json (51KB)
**Alternative validation rules** - similar structure to odisha_layers.json but for a different jurisdiction or use case.
- Used by utility scripts to clean up CAD-to-PDF mappings
- Has slightly different layer definitions

## CAD-to-PDF Configuration

### ppa_cadtopdf.json (28KB)
Configuration for CAD-to-PDF conversion layer visibility.
- Contains layer visibility settings for different drawing sheets
- Used by utility scripts (analyze_layers.py, correct_layers.py)
- **Not used by the main application**

### ppa_cadtopdf_corrected.json (20KB)
Cleaned version of ppa_cadtopdf.json.
- Generated by correct_layers.py
- Contains only layers that match patterns in ppa_layers.json
- **Generated file - safe to delete**

### odisha_cadtopdf.json (28KB)
Similar to ppa_cadtopdf.json but for Odisha jurisdiction.
- Layer visibility configuration for CAD-to-PDF conversion
- **Not used by the main application**

## Summary

**Used by main app:**
- `odisha_layers.json` - Master validation rules

**Used by utility scripts:**
- `ppa_layers.json` - Alternative rules
- `ppa_cadtopdf.json` - Layer visibility config

**Generated files (safe to delete):**
- `ppa_cadtopdf_corrected.json` - Output of correct_layers.py

## Adding New Rules

To add or modify validation rules:
1. Edit `odisha_layers.json` directly, or
2. Use the admin UI at `http://localhost:8080/admin`

The admin UI validates JSON syntax before saving.
